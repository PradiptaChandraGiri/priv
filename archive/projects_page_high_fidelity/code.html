Distributed Task Scheduler

This project involved designing and implementing a robust distributed task scheduling framework from scratch. The core objective was to manage and execute asynchronous background jobs across a cluster of commodity machines with high reliability and zero single points of failure.

Technical Highlights

  • Implemented a custom Raft consensus module to handle leader election and log replication, ensuring cluster state consistency even during network partitions.
  • Designed a gRPC-based communication protocol for low-latency dispatching of task payloads between the leader and worker nodes.
  • Built a persistent storage layer using embedded Key-Value stores (BadgerDB) to guarantee exactly-once execution semantics.
